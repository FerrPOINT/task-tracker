# Internationalization (i18n) — Task Tracker

## 1. Overview

Приложение поддерживает русский (`ru`) и английский (`en`) языки. По умолчанию — русский. Локаль пользователя хранится в профиле и в cookie/localStorage.

## 2. Libraries

- Backend: встроенные Rust локали через `fluent` или статические файлы YAML.
- Frontend: `react-i18next` 15.5.0 + `i18next` 25.1.0 + `i18next-http-backend`.

## 3. Supported Locales

| Locale | Language | Fallback |
|--------|----------|----------|
| `ru` | Русский | `en` |
| `en` | English | — |

## 5. Namespace Convention

| Namespace | Content |
|-----------|---------|
| `common` | Buttons, labels, navigation, generic terms |
| `auth` | Login, register, password reset |
| `issue` | Issue fields, actions, statuses |
| `project` | Project settings, roles |
| `board` | Kanban/scrum terms |
| `notifications` | Notification events, templates |
| `errors` | Error codes and messages |

## 7. Usage in React

```tsx
import { useTranslation } from "react-i18next"

export function IssueForm() {
  const { t } = useTranslation(["issue", "common"])
  return (
    <form>
      <label>{t("issue.summary")}</label>
      <Button>{t("common.save")}</Button>
    </form>
  )
}
```

## 8. Interpolation

```json
{
  "issueAssigned": "Задача {{issueKey}} назначена {{assigneeName}}"
}
```

```tsx
<p>{t("notifications.issueAssigned", { issueKey: "PROJ-123", assigneeName: "Alice" })}</p>
```

## 9. Pluralization

```json
{
  "comments": {
    "one": "{{count}} комментарий",
    "few": "{{count}} комментария",
    "many": "{{count}} комментариев"
  }
}
```

```tsx
<p>{t("issue.comments", { count: 5 })}</p>
```

## 10. Date/Number Formatting

```tsx
import { format } from "date-fns"
import { ru } from "date-fns/locale"

const localeMap = { ru, en: undefined }
format(date, "dd MMM yyyy", { locale: localeMap[i18n.language] })
```

## 11. Backend Localization

- Email templates: язык из профиля получателя.
- Error messages: `Accept-Language` header или fallback.
- Notification titles/bodies: формируются на языке получателя.

## 12. Locale Switching

```tsx
function LocaleSwitcher() {
  const { i18n } = useTranslation()
  return (
    <Select value={i18n.language} onValueChange={i18n.changeLanguage}>
      <SelectItem value="ru">Русский</SelectItem>
      <SelectItem value="en">English</SelectItem>
    </Select>
  )
}
```

## 13. Adding a New Locale

1. Создать `public/locales/{locale}/`.
2. Скопировать структуру из `en`.
3. Перевести.
4. Добавить в `shared/i18n/config.ts`.
5. Обновить `SupportedLocale` type.

## 14. Testing i18n

- Все пользовательские строки вынесены в JSON.
- Тесты используют `i18n.changeLanguage("en")`.
- Проверка отсутствия raw-строк в JSX.

## 15. RTL (future)

В MVP не поддерживается. Если понадобится — добавить `dir="rtl"` и mirrored layout tokens.

## 16. Server-Side Rendering

Если введём SSR:

- определять locale по cookie / header;
- загружать ресурсы на сервере;
- встраивать `window.__I18N__`.
